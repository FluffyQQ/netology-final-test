## Описание

Поле «Сумма перевода» позволяет ввести отрицательное число. Из-за этого комиссия тоже становится отрицательной, а условие проверки доступных средств всегда выполняется. В итоге кнопка «Перевести» появляется и банк «принимает перевод -100 ₽».

Это критический баг — в реальной системе такое могло бы привести к зачислению денег вместо списания.

## Шаги воспроизведения

1. Запустить сервис: `python3 -m http.server 8000` из папки `dist/`
2. Открыть `http://localhost:8000/?balance=30000&reserved=20001`
3. Кликнуть на карточку «Рубли»
4. Ввести номер карты: `1111222233334444`
5. В поле «Сумма перевода» ввести `-100`
6. Посмотреть на комиссию и кнопку «Перевести»

## Ожидаемый результат

Минус игнорируется (остаётся `100`) или показывается сообщение об ошибке. Кнопка «Перевести» **не появляется**.

## Фактический результат

- В поле сохраняется `-100`
- Комиссия показывается как `-10 ₽`
- Кнопка «Перевести» зелёная и активная
- При нажатии появляется alert: `Перевод -100 ₽ на карту ... принят банком!`

## Причина бага (анализ кода)

В файле `dist/assets/index-BUH56GOL.js`:

```js
// обработчик намеренно сохраняет знак минус
let L = R.target.value.startsWith("-"),
    j = R.target.value.replace(/\D/g, "");
L && (j = "-" + j);   // ← минус возвращается обратно
p(Number(j));          // O = -100

// при O=-100 комиссия тоже отрицательная
const h = Math.floor(O / 100) * 10;   // = -10

// проверка всегда выполняется: 30000-20001+10+100 = 10109 > 0
const A = o - s - h - O > 0;
```

## Как исправить

Удалить строку `L && (j = "-" + j)` из обработчика поля суммы.  
Дополнительно добавить проверку `O > 0` перед расчётом условия `A`.
