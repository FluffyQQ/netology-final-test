## Описание

По условиям задачи: «Если сумма перевода с учётом комиссии **не превышает** доступную сумму, перевод можно осуществить». Значит перевод должен быть возможен и когда сумма + комиссия **равна** доступному остатку.

Но в коде стоит строгое `>` вместо `>=`, поэтому граничный случай обрабатывается неверно.

## Шаги воспроизведения

1. Запустить сервис: `python3 -m http.server 8000` из папки `dist/`
2. Открыть `http://localhost:8000/?balance=1100&reserved=0`
   - Доступно: 1100 - 0 = 1100 ₽
3. Кликнуть на «Рубли»
4. Ввести номер карты: `1111222233334444`
5. В поле суммы ввести `1000`
   - Комиссия: 100 ₽
   - Итого: 1000 + 100 = 1100 ₽ = доступный остаток

## Ожидаемый результат

Кнопка «Перевести» **отображается** — сумма с комиссией не превышает остаток.

## Фактический результат

❌ Показывается «Недостаточно средств на счете», кнопки нет.

## Причина бага (анализ кода)

```js
// строгое > вместо >=
const A = o - s - h - O > 0;
// при balance=1100, reserved=0, commission=100, amount=1000:
// 1100 - 0 - 100 - 1000 = 0 → 0 > 0 = false → кнопки нет
```

По условию ТЗ должно быть `>= 0`.

## Как исправить

Заменить `> 0` на `>= 0` в условии переменной `A`.
