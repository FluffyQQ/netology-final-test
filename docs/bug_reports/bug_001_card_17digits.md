## Описание

Поле «Номер карты» позволяет ввести 17 цифр вместо ровно 16. При этом форма ввода суммы всё равно открывается. По условиям задачи номер карты должен состоять из **ровно 16 цифр**.

## Шаги воспроизведения

1. Запустить сервис: `python3 -m http.server 8000` из папки `dist/`
2. Открыть `http://localhost:8000/?balance=30000&reserved=0`
3. Кликнуть на карточку «Рубли»
4. В поле «Номер карты» ввести 17 цифр: `12345678901234567`
5. Посмотреть, открылась ли форма «Сумма перевода»

## Ожидаемый результат

17-й символ не принимается (обрезается или не вводится). Форма «Сумма перевода» **не открывается** при 17 цифрах.

## Фактический результат

17 цифр сохраняются в поле (отображаются как `1234 5678 9012 3456 7`). Форма «Сумма перевода» открывается — пользователь может совершить перевод с несуществующим номером карты.

## Причина бага (анализ кода)

В файле `dist/assets/index-BUH56GOL.js` в обработчике поля карты:

```js
// обрезает до 17 — должно быть до 16
L.length > 17 && (L = L.slice(0, 17));

// форма открывается при >= 16, а не === 16
y.length >= 16 && P.jsxs(...)
```

## Как исправить

- Заменить `L.length > 17` на `L.length > 16`
- Заменить условие `y.length >= 16` на `y.length === 16`
